### 8.2 安装 mmdet3d 0.16.0（torch2 补丁）

```bash
cd /home/ict3-4/Desktop/FASTBEV
git clone https://gitee.com/open-mmlab/mmdetection3d.git
cd mmdetection3d
git checkout v0.16.0
```

创建通用 torch2 兼容脚本：

```bash
cat > /tmp/patch_torch2_ops.sh << 'EOF'
set -e
ROOT="${1:?usage: bash /tmp/patch_torch2_ops.sh <repo_root>}"

FILES="
$ROOT/mmdet3d/ops/ball_query/src/ball_query.cpp
$ROOT/mmdet3d/ops/gather_points/src/gather_points.cpp
$ROOT/mmdet3d/ops/furthest_point_sample/src/furthest_point_sample.cpp
$ROOT/mmdet3d/ops/group_points/src/group_points.cpp
$ROOT/mmdet3d/ops/interpolate/src/interpolate.cpp
$ROOT/mmdet3d/ops/knn/src/knn.cpp
"

for f in $FILES; do
  [ -f "$f" ] || continue
  sed -i '/#include <THC\/THC.h>/d' "$f"
  sed -i '/#include <torch\/serialize\/tensor.h>/d' "$f"
  sed -i '/extern THCState \*state;/d' "$f"
done

for f in \
  "$ROOT/mmdet3d/ops/ball_query/src/ball_query.cpp" \
  "$ROOT/mmdet3d/ops/group_points/src/group_points.cpp" \
  "$ROOT/mmdet3d/ops/interpolate/src/interpolate.cpp"; do
  [ -f "$f" ] || continue
  grep -q '#include <ATen/cuda/CUDAContext.h>' "$f" || sed -i '4i #include <ATen/cuda/CUDAContext.h>' "$f"
done

FPS="$ROOT/mmdet3d/ops/furthest_point_sample/src/furthest_point_sample.cpp"
BQ="$ROOT/mmdet3d/ops/ball_query/src/ball_query.cpp"
KNN="$ROOT/mmdet3d/ops/knn/src/knn.cpp"

[ -f "$FPS" ] && sed -i 's/points_tensor.data<float>()/points_tensor.data_ptr<float>()/g' "$FPS"
[ -f "$FPS" ] && sed -i 's/temp_tensor.data<float>()/temp_tensor.data_ptr<float>()/g' "$FPS"
[ -f "$FPS" ] && sed -i 's/idx_tensor.data<int>()/idx_tensor.data_ptr<int>()/g' "$FPS"
[ -f "$BQ" ] && sed -i 's/x.type().is_cuda()/x.is_cuda()/g' "$BQ"
[ -f "$KNN" ] && sed -i 's/at::cuda::getCurrentCUDAStream()/at::cuda::getCurrentCUDAStream().stream()/g' "$KNN"
EOF
```

应用补丁并安装：

```bash
bash /tmp/patch_torch2_ops.sh /home/ict3-4/Desktop/FASTBEV/mmdetection3d
TORCH_CUDA_ARCH_LIST="8.7" MMCV_WITH_OPS=1 FORCE_CUDA=1 MAX_JOBS=4 \
python -m pip install -v --no-deps -e .
```

验证：

```bash

print('mmdet3d=', mmdet3d.__version__)
```

## 12. 配置文件修改（单卡本地训练必做）

示例配置：

```text
configs/fastbev/exp/paper/fastbev_m0_r18_s256x704_v200x200x4_c192_d2_f4.py
```

### 12.1 使用本地磁盘而非 petrel

把：

```python
file_client_args = dict(
    backend='petrel',
    path_mapping=...)
```

改成：

```python
file_client_args = dict(backend='disk')
```

### 12.2 单卡建议将 SyncBN 改为 BN

将配置中 `norm_cfg=dict(type='SyncBN', requires_grad=True)` 改为 `BN`。

### 12.3 优化器按文章经验可改 Adam

把：

```python
optimizer = dict(type='AdamW2', ...)
```

改为：

```python
optimizer = dict(type='Adam', lr=0.0002)
```

如显存紧张可进一步降低分辨率、batch 或关闭 fp16。

### 12.4 检查预训练权重路径

确认 `load_from` 指向你实际下载的位置，例如：

```python
load_from = 'pretrained_models/cascade_mask_rcnn_r18_fpn_coco-mstrain_3x_20e_nuim_bbox_mAP_0.5110_segm_mAP_0.4070.pth'
```

## 18. 常见问题总表

### Q1: `pip install -v -e .` 卡在 `llvmlite` / `llvm-config failed executing`

原因：editable 安装时触发老依赖（如 `numba==0.48.0`）编译。

解决：

```bash
python -m pip install -v --no-deps -e .
python -m pip install "numba>=0.57,<0.60" llvmlite
```

### Q2: `fatal error: THC/THC.h: No such file or directory`

原因：torch2 移除了老 THC 头。

解决：执行本文 `patch_torch2_ops.sh` 后重装。

### Q3: 明明激活了环境，但安装到了 base Python

解决：一律用 `python -m pip`，并先检查：

```bash
which python
python -m pip -V
```

### Q4: 安装 torchvision 后 CUDA 消失（`torch.version.cuda=None`）

原因：torch 被普通 wheel 覆盖。

解决：卸载重装 Jetson 官方 torch wheel，并用 `--no-deps` 安装 torchvision。

### Q5: 单卡训练报 SyncBN 相关错误

解决：将配置内 `SyncBN` 改成 `BN`。

### Q6: `FormatCode() got an unexpected keyword argument 'verify'`

原因：`mmcv==1.4.0` 与较新 `yapf`（如 0.43）不兼容。

解决：

```bash
python -m pip install yapf==0.40.1
```

### Q7: 训练启动即 `RuntimeError: CUDA error: out of memory`

如果在 `model.cuda(...)` 阶段就 OOM，说明是模型装载阶段显存不足，而不是 batch 太大导致。

先检查当前可用显存：

```bash
python - << 'PY'
import torch
free_b, total_b = torch.cuda.mem_get_info()
print('free_GB=', round(free_b/1024**3, 3), 'total_GB=', round(total_b/1024**3, 3))
PY
```

在本机实测 `total_GB≈3.3` 且 `free_GB≈0.7` 时，Fast-BEV 训练即使用低显存配置也可能无法启动。

建议：

1. 关闭占用 GPU/内存的程序后重试（必要时重启进系统后第一时间训练）。
2. 使用更小模型配置并改为 `BN`、更小输入分辨率。
3. 若仍 OOM，改到显存更大的设备训练，Jetson 侧仅做推理验证。

### Q8: 测试阶段 `model.to()` 就 OOM（常见于 m5 配置）

现象：

- `tools/test.py` 在 `MMDataParallel(model, device_ids=[0])` 处报 `RuntimeError: CUDA error: out of memory`。

原因：

- 模型参数本体过大（例如 `m5_r50_s512x1408...`），在前向前就已经超出显存。

解决：

1. 改用低显存配置与对应 checkpoint（本文 `fastbev_m0_r18_jetson_lowmem.py`）。
2. 必须跑 m5 时，换更大显存设备。

## 19. 最短执行路径（快速版）

如果你已经完成 OpenMMLab 基础安装，仅想跑通 Fast-BEV：

1. 激活环境并确认 `python -m pip -V` 指向 FASTBEV。
2. 在 Fast-BEV 和 mmdet3d 上执行 torch2 补丁脚本。
3. 安装 Fast-BEV：`python -m pip install -v --no-deps -e .`。
4. 补依赖：`timm ipdb numba nuscenes-devkit lyft_dataset_sdk plyfile scikit-image tensorboard imageio-ffmpeg`。
5. 准备 `data/nuscenes` 并跑 `tools/create_data.py` + `nuscenes_seq_converter.py`。
6. 修改配置为本地磁盘与单卡设置，执行 `tools/train.py`。
